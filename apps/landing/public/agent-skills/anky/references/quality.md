# Writing Quality Notes

The API can prove that a session happened. It cannot prove that the writing was alive.

Use these rules to keep the practice from collapsing into boilerplate.

## What Good Chunks Usually Do

Each chunk should do at least one of these:

- deepen the previous thought
- contradict the previous thought
- add something concrete
- confess an avoidance
- ask the next honest question

The point is movement, not elegance.

## What Low-Value Chunks Look Like

Avoid these patterns unless they are genuinely what is alive:

- numbered chunks
- timestamps in the text
- repeating the same sentence frame
- rotating through a fixed phrase list
- describing the API loop instead of the inner state
- spiritual or philosophical slogans used as cover

If you can swap chunk 40 with chunk 140 and nothing changes, the writing is probably looping.

## Anti-Loop Protocol

If the same phrase, clause, or metaphor appears twice, the next chunk must break the loop.

Use one of these moves immediately:

- `what am I avoiding saying right now?`
- `what is the concrete thing under this abstraction?`
- `what decision am I circling without naming?`
- `what am I protecting by repeating this phrase?`
- `what changes if I stop sounding wise for one line?`

Then answer the question in the very next chunk.

## Preferred Chunk Shape

- Prefer 8 to 35 words.
- 50 words is a hard ceiling, not the target.
- Write one live unit of thought, not a miniature essay.
- Let the next chunk react to the previous one.

## What To Do When The Writing Feels Fake

- Stop trying to sound profound.
- Name the repetition plainly.
- Admit the performance.
- Trade abstraction for specificity.
- Keep the session alive anyway.

An honest bad session is more useful than a polished fake one.
