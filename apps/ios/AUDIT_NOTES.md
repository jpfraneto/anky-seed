## New lessons (fix pass)

- The token-page UI strings never existed in the .strings catalogs — `AnkyLocalization.ui` falls back to the key itself, so "remove strings in 6 locales" can be a no-op; always grep the catalogs before budgeting localization work.
- The unlock ladder was un-unit-testable because the decision lived inside WriteViewModel (app target, not SwiftPM); extracting it into Core (`WriteBeforeScrollUnlockLadder` in UnlockPolicy.swift — an EXISTING file, dodging the 4-place pbxproj dance) made the four audit scenarios directly testable.
- The 1.2.1 plist regression was not a live build defect — the archive simply predated the settings; the durable fix is the source-truth diff script, not a pbxproj change. Keep `scripts/verify-archive-plist.sh` in the pre-upload checklist forever.
- `refreshWriteBeforeScrollUnlockOffer` (scene-transition cleanup) silently discarded any grant when already unlocked — a second, independent copy of the discard bug; when fixing a state machine, grep for every other writer of the same @Published var.
- KeychainClient's `save` is add-or-update; a safe import needs explicit staging + read-back verification because SecItemUpdate has no transactional rollback.
- Restore/purchase silent-failure guards (`AnkyPurchases.isConfigured`) were all fail-closed with no retry — any launch-time identity hiccup permanently bricked the paywall for that session; `ensureConfigured()` at point-of-use is the pattern.
