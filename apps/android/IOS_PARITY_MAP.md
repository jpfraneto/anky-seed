# iOS to Android Parity Map

This file is the working inventory for `ANKY_ANDROID_SWIFT_PARITY_GOAL.md` and the delta brief in `ANKY_ANDROID_DELTA_PARITY_GOAL.md`. iOS remains the canonical source in `apps/ios/Anky`.

## Current Delta Status

- Reveal primary flow now matches the current iOS inline rhythm: reconstructed writing, privacy divider, inline saved/streaming/error reflection content, and a bottom gold action button.
- Android no longer uses the previous bottom conversation/sheet as the primary Reveal reflection surface.
- Reveal copy is section-aware (`copy writing`, `copy reflection`) and reflection copy includes title plus body.
- Reveal delete confirmation copy now matches iOS exactly and remains local-only.
- Saved `creditsRemaining` is persisted and rendered truthfully when present; Android invalidates the credit cache only after responses with non-null `creditsRemaining`.
- Android now mirrors iOS per-account reflection credit caching across Reveal and You, including claimed-free-reflection state, cached initial balance display, You on-entry cache refresh, unspent-gift purchase locking, and write-back after mirror success, refresh, purchase, and restore paths.
- Write live input now rejects multi-glyph paste/replacement mutations. Explicit `.anky` import remains separate from writing input.
- Android trial proof remains intentionally disabled until a Play Integrity/device recall design exists with server-side verification.
- You home has been refreshed against the current Swift panel shape: Android now shows Data, Credits, Support / Feedback, Privacy Policy, Terms & Conditions, and `$ANKY on Base` with native Android handling for readable writing export, backup/restore, credits, mailto support, legal/privacy detail pages, and direct contract copy.
- Android You now includes the current iOS all-ankys local history route from the tappable stats panel, backed by local complete session summaries and Reveal navigation by hash.
- Android You now includes the current iOS hidden exclamation entry for destructive account/data deletion in native Android form, clearing local writing/reflection/index/draft/request/first-open/settings/identity/RevenueCat/credit-cache state and resetting root Write/Map/credit badge state.
- Android You now exposes the app/device-lock toggle on the main panel like iOS, with Android-native biometric/device credential capability checks and authenticated enable.
- Android You now uses a small `$ANKY` row symbol for the `$ANKY on Base` menu item like iOS' `you-icon-anky-token` / `dollarsign.circle`, while keeping the migrated coin bitmap for the token detail page.
- Android You home no longer mounts the stale large `you` title or visible stats panel; the first screen now starts with the current Swift-shaped main row panel.
- Android You home now includes compact inline `You` title chrome with the destructive exclamation control at top-right, matching iOS' inline navigation title shape without restoring the stale large title.
- Android You `Credits` row now opens the current iOS-shaped reflection credits sheet instead of navigating to a full detail page, while retaining the credits detail route for native deep links and Reveal handoff; the sheet includes the current iOS native refresh icon treatment and long-press copy / external AI fallback footer.
- Android Write terminal-silence completion now uses an iOS-shaped pending Map-to-Reveal handoff, and Reveal `CONTINUE` on an incomplete sealed fragment reopens that writing directly in Write by trimming terminal silence, freezing until the next glyph, and replacing the old fragment when saved again.
- Android `.anky` duration/completion semantics now match current Swift and shared TypeScript: terminal silence is parsed and preserved, but `durationMs` is writing duration only and completion is `writingDurationMs >= 480000`.
- Android now has the current iOS first-launch onboarding flow using the three migrated onboarding images, matching page copy, CTA labels, dots, and final Write portal handoff.
- Android now has resource-backed system-locale strings for first-launch onboarding lines, all three onboarding CTAs, the onboarding accessibility label, root tab labels, the Write launch bubble, ritual steps, writing-time/top-chrome/import labels, the device-lock activation prompt, app-lock biometric prompt reasons, You recovery/backup/encrypted-backup biometric prompt reasons, the main You home row panel/title/status labels, You stats/history/device-lock/delete-toggle labels, the highest-traffic You Account/Data detail-page copy including account clipboard labels, daily-reminder settings labels plus iOS-matched notification constants, destructive Data/account dialogs, and backup/share actions, retained Privacy/Terms page headers, retained Credits detail rules/actions/support note, retained Anky Experience overlay labels/copy/accessibility, the recovery import dialog, the primary You companion prompt action and close labels, top-level Reveal action/delete/reflection CTA labels, Reveal conversation action/status/invitation/short-session labels, Reveal/tag navigation labels including tag empty-state and word-count labels, Reveal copy/copy-confirmation/privacy/live-reflection/progress/error copy inventory, reflection credits sheet labels, and Map shell/accessibility labels across the same shipped iOS locale set available in this checkout: English, Spanish, French, German, Hindi, and Simplified Chinese. Retained Android-only debug/local-tool labels, long legal article bodies, and several platform-specific secure-storage strings intentionally remain English where current iOS does not localize those exact keys or uses iCloud-specific wording.
- Android Data now has a local encrypted recovery-phrase backup path in addition to readable exports, bringing it closer to the shipped iOS backup/recovery expectation while staying Android-local rather than iCloud-backed.
